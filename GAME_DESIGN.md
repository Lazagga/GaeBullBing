# 개불빙 통합 게임플레이 기획서

> 이 문서는 현재 확정된 턴 구조, 전투 우선순위, 상태 효과, 특수 몬스터, JSON 데이터 운영 규칙을 통합한 기준 문서다.

# 1. 기본 용어

- **골인 지점**: 몬스터가 한 바퀴 이동한 뒤 도달하는 시작 타일이다.
- **골인 근접순**: 누적 이동 거리가 큰 순서다.
- **타워 최우선 대상**: 해당 타워가 이전 공격에서 유지한 대상 등 타워 고유 규칙으로 지정된 대상이다.
- **페이즈 개시/종료**: 해당 시점에 등록된 효과가 있을 때 실행하는 논리적 훅이다. 등록된 효과가 없으면 별도 처리 없이 통과한다.
- **작성 순서**: 같은 페이즈 안에 기재된 효과는 문서의 위에서 아래 순서로 처리한다.

# 2. 공통 실행 원칙

1. 별도 규칙이 없으면 몬스터와 타일은 골인 지점에 가까운 순서로 처리한다.
2. 같은 페이즈의 효과는 문서 작성 순서대로 처리한다.
3. 앞선 대상의 처리 결과는 다음 대상에게 즉시 반영한다.
4. 사망한 몬스터는 이후 효과와 대상 선정에서 제외한다.
5. 게임 규칙 계산은 Core에서 수행하고, 화면 연출은 계산 결과를 순서대로 재생한다.

# 3. 타워 처리 순서

타워는 골인 지점에 가까운 타일부터 순차 처리한다.

각 타워는 다음 단계를 모두 마친 뒤 다음 타워로 넘어간다.

1. 타워 스탠바이
2. 공격 대상 인식
3. 타워 공격
4. 공격 종료 처리

따라서 뒤 타워는 앞 타워가 발생시킨 사망, 넉백, 상태 이상, 타일 효과가 반영된 전장을 기준으로 대상을 다시 인식한다.

## 3.1 타워 공격 대상 우선순위

사거리 내 후보를 다음 순서로 정렬한다.

1. 타워 어그로 효과가 있는 몬스터
2. 타워의 최우선 대상
3. 보스
4. 골인 지점에 가까운 몬스터
5. 현재 체력이 높은 몬스터
6. 모든 조건이 같으면 먼저 생성된 몬스터

상위 조건이 다르면 하위 조건은 비교하지 않는다.

# 4. 전체 턴 순서

1. 턴 시작 페이즈
2. 주사위 페이즈
3. 플레이어 이동 페이즈
4. 건설 및 업그레이드 페이즈
5. 몬스터 스탠바이 페이즈
6. 몬스터 스폰 및 이동 페이즈
7. 타워 처리 페이즈
8. 타일 계산 페이즈
9. 턴 종료 페이즈

# 5. 턴 시작 페이즈

## 개시

- 현재 턴 번호를 확정한다.
- 이번 턴에 적용할 임시 상태를 준비한다.

## 본체

- 턴 시작 시 발동하는 효과와 시스템 이벤트를 처리한다.

## 종료

- `Player Turn` 배너를 표시한 뒤 제거한다.

# 6. 주사위 페이즈

## 개시

- `ROLL` 버튼을 활성화한다.

## 본체

- 플레이어 입력을 기다린다.
- 입력 후 각 주사위 결과를 결정한다.
- 주사위 연출을 재생한다.

## 종료

- 최종 주사위 눈과 합계를 확정한다.

# 7. 플레이어 이동 페이즈

## 개시

- 주사위 합계를 최종 이동 거리로 확정한다.

## 본체

- 플레이어 이동 상태는 최종 위치를 기준으로 계산한다.
- 화면에서는 한 타일씩 이동하는 연출을 재생한다.
- 현재 기획에서는 플레이어의 중간 이동 경로에 별도 타일 효과를 적용하지 않는다.
- 한 바퀴를 완주했다면 완주 보상을 처리한다.
- 완주할 때마다 모든 타워 공격력이 영구적으로 5% 증가한다.

## 종료

- 최종 도착 타일을 확정한다.
- `이 타워를 밟을 경우`, `이 타일을 밟을 경우` 효과를 적용한다.

# 8. 건설 및 업그레이드 페이즈

## 개시

- 도착 타일의 건설 및 업그레이드 가능 여부를 확인한다.
- 현재 타워, 원소, 티어에 맞는 선택지 풀을 생성한다.
- Build/Upgrade UI를 표시한다.

## 본체

- 플레이어 선택을 기다린다.
- 선택한 타워를 건설하거나 업그레이드를 적용한다.
- JSON에 정의된 스탯 변경, 효과 ID, 효과 수치를 런타임 타워에 복사한다.
- 선택 UI를 제거한다.

## 종료

- 최종 타워 상태를 확정하고 건설 또는 업그레이드 연출을 마친다.

# 9. 몬스터 스탠바이 페이즈

몬스터는 골인 근접순으로 처리한다.

## 개시

- 기존 상태 이상을 점검한다.

## 본체

1. 이전 턴에 한 번 이상 넉백된 몬스터가 영구 넉백 면역을 획득한다.
2. 기존 화상 스택에 따른 화상 피해를 적용한다.
3. 생존했다면 현재 위치가 불 타일일 때 화상 1스택과 불 피해를 적용한다.

현재 위치의 불 타일이 새로 부여한 화상 스택은 다음 화상 피해 계산부터 기존 화상으로 취급한다.

## 종료

- 빙결 카운터가 1 이상이면 이번 이동력을 0으로 확정하고 빙결 카운터를 1 차감한다.
- 기절 등 이동 제한 효과를 적용한다.
- 출발 타일이 빙판이면 빙판 이동 제한을 적용한다.
- 1차 이동속도를 확정한다.

# 10. 몬스터 스폰 및 이동 페이즈

- 기존 몬스터를 골인 근접순으로 처리한다.
- 이번 턴 신규 몬스터는 실행 목록의 마지막에 배치한다.
- 내부 객체 생성 시점은 달라도 되지만 실제 이동 및 연출 순서는 기존 몬스터 뒤여야 한다.

## 지상 몬스터

1. 확정된 이동력만큼 한 칸씩 이동한다.
2. 진입한 각 타일의 효과를 처리한다.
3. 빙판 진입 시 남은 이동을 제한한다.
4. 벽 타워 진입 시 즉시 벽 피해를 받고 이동을 종료한다.
5. 불 타일 진입 시 즉시 화상 1스택과 불 피해를 받는다.
6. 골인 도달 여부와 최종 위치를 확정한다.

## 비행 몬스터

- 출발 타일에서 목적지 타일까지 직접 이동한다.
- 중간에 지나가는 타일에는 진입하지 않은 것으로 처리한다.
- 출발 타일 효과는 몬스터 스탠바이에서 적용한다.
- 목적지 타일 효과는 도착 시 적용한다.
- 이동력이 0이면 출발지와 목적지가 같으므로 같은 타일 효과를 중복 적용하지 않는다.

# 11. 까마귀 보스

까마귀는 비행 이동을 사용하는 보스다.

## 이동 예시

0번에서 4번 타일로 이동할 경우:

- 0번 타일 효과 적용
- 1, 2, 3번 타일 효과 무시
- 4번 타일 효과 적용

## 상태 효과

- 넉백에만 면역이다.
- 빙결, 화상, 감전, 기절 등 다른 상태 이상은 정상적으로 적용된다.
- 빙결 상태라면 일반 몬스터와 동일하게 해당 이동이 차단된다.
- 출발 타일이 빙판이면 이동 제한을 받는다.
- 목적지의 불 타일과 벽 타워 피해를 받는다.
- 중간 경로의 불, 얼음, 벽 효과는 받지 않는다.

## 깃털

- 깃털은 지정된 타워의 공격을 봉인한다.
- 깃털 봉인 중에도 해당 타워의 쿨다운은 정상적으로 감소한다.
- 쿨다운이 0인 상태에서 깃털이 풀리면 해당 타워는 바로 공격할 수 있다.
- 실제 공격 대상이 없어 공격하지 않았다면 새 공격 쿨다운을 설정하지 않는다.

## 꼭짓점 속성 강화

- 9번 또는 27번 꼭짓점 타일에서 선택한 속성의 모든 타워 공격력을 영구적으로 20% 증가시킨다.
- 이후 건설되는 같은 속성 타워에도 적용한다.

# 12. 타워 스탠바이

1. 쿨다운을 감소시킨다. 깃털 봉인 여부와 관계없이 감소한다.
2. 이번 턴 적용할 타워 버프를 반영한다.
3. 깃털, 쿨다운, 공격 불가 효과를 확인한다.
4. 최종 공격력, 사거리, 대상 수, 공격 횟수를 확정한다.

벽 타워와 공격 대신 지속 효과만 제공하는 타워는 일반 공격 대상에서 제외한다.

## 공격력 계산

공격력은 아래 순서로 계산한다.

1. 기본 공격력에 일반 업그레이드의 공격력 증가량을 더한다.
2. 업그레이드 배율과 9·27번 타일 효과, 맵 완주 효과 등의 배율을 곱한다.
3. 풀 업그레이드 이후 추가 업그레이드로 얻은 공격력은 마지막에 더한다.

공식: `(기본 공격력 + 업그레이드 공격력) × 배율 + 풀 업그레이드 이후 추가 공격력`

예시: `(15 + 120) × 150% + 60`

타일 정보에는 최종 공격력과 함께 위 계산식을 현재 수치로 표시한다.

# 13. 타워 공격

1. 공격 대상을 인식한다.
2. 공격을 실행한다.
3. 적중 효과를 데이터에 기록된 순서로 적용한다.
4. 넉백이 발생하면 한 칸 뒤로 이동시킨다.
5. 넉백 도착 타일의 효과를 적용한다.
6. 공격 종료 후 턴 단위 타워 버프 지속시간을 1 차감한다.

## 넉백

- 한 번의 넉백 거리는 1칸이다.
- 넉백 면역을 아직 획득하지 않았다면 같은 턴에 여러 번 넉백될 수 있다.
- 한 번 이상 넉백된 몬스터는 다음 몬스터 스탠바이에 영구 넉백 면역을 획득한다.
- 넉백 도착지가 불 타일이면 화상 1스택과 불 피해를 받는다.
- 넉백 도착지에 벽 타워가 있으면 벽 피해를 받는다.

## 벽 타워

- 일반 타워 공격 페이즈에는 공격하지 않는다.
- 지상 몬스터가 이동 중 벽 타워 타일에 진입하면 즉시 피해를 주고 이동을 종료시킨다.
- 비행 몬스터가 목적지로 벽 타워 타일에 도착하면 피해를 준다.
- 넉백으로 벽 타워 타일에 도착하면 피해를 준다.

## 굴러가는 돌

- 굴러가는 돌은 타워 공격의 한 종류다.
- 해당 돌 타워의 공격 순서에 즉시 실행한다.
- 돌의 전체 이동과 공격을 마친 뒤 다음 타워를 처리한다.
- 돌의 공격 결과로 발생한 사망, 넉백, 상태 이상은 뒤 타워의 대상 인식에 반영된다.
- 돌에 넉백이 적용되어 있으면 충돌 즉시 몬스터를 한 칸 민 뒤 돌도 다음 칸으로 이동한다.
- 밀린 몬스터와 돌이 다음 칸에서 다시 만나면 다시 피해와 넉백을 적용하며, 돌의 이동이 끝날 때까지 반복한다.

## 공격 형식 변경과 부가효과

- `explode`, `chain_line`, `rolling_stone`은 기본 투사체 공격을 대체하는 공격 형식이다.
- 공격 형식이 변경되면 기본 투사체 공격은 별도로 발생하지 않는다.
- 화상, 빙결, 감전, 넉백, 타일 전염, 상태 이상 전염 등의 적중 부가효과는 변경된 공격에 맞은 모든 대상에게 적용한다.
- `chain_line`과 `chain_tile`을 함께 보유하면 먼저 라인 전체를 공격하고, 라인 공격에 맞은 각 대상을 시작점으로 `chain_tile`을 각각 적용한다.
- `burn_explode`는 해당 공격의 화상 적용까지 끝난 시점에 대상의 화상이 10스택 이상이면 발동한다.
- `burn_explode`가 발동하면 대상 타일과 앞뒤 인접 타일, 총 3칸을 공격하며 각 피격 대상에게 적중 부가효과를 적용한다.

# 14. 타일 계산 페이즈

## 개시

- 보드의 타일 상태를 점검한다.

## 본체

- 불 타일과 얼음 타일의 지속시간을 1 차감한다.
- 서로 독립적인 지속시간 감소는 일괄 처리할 수 있다.
- 타일 간 결과가 서로 영향을 주는 효과는 골인 근접순으로 처리한다.

## 종료

- 지속시간이 0이 된 타일 효과를 제거하고 기본 상태로 복원한다.

# 15. 턴 종료 페이즈

## 개시

- 턴 전체 결과를 점검한다.

## 본체

- 턴 단위 임시 상태를 정리한다.
- 다음 턴에 사용할 데이터를 준비한다.
- 난이도 진행, 처치 수, 플레이어 보상을 반영한다.

## 종료

- 현재 턴을 종료한다.
- 다음 턴 번호로 증가시킨 뒤 턴 시작 페이즈로 이동한다.

# 16. 몬스터 처치 보상

- 몬스터 처치 시 `Monster.json`의 `kill_rewards.dice_points`를 플레이어 데이터에 누적한다.
- 같은 몬스터의 사망 결과가 여러 효과 결과에 포함되어도 보상은 한 번만 지급한다.
- 현재 주사위 포인트는 플레이어 데이터에 누적만 하며 별도 소비 기능은 추후 구현한다.

# 17. JSON 데이터 운영 규칙

JSON은 기획 데이터의 원본이다.

## `Pattern.json`

- 런타임 시작 시 직접 읽는다.
- `required_kills`, `multiplier`, `defense_per_wave`, `level`, `spawn_pattern`을 사용한다.
- 실행 중 수정한 값은 새 게임 또는 런타임 재시작 후 반영한다.

## `Monster.json`

- Editor importer가 `MonsterDefinition`으로 변환한다.
- 사용 필드:
  - `id`
  - `name`
  - `tier`
  - `appearance_wave`
  - `base_stats.max_hp`
  - `base_stats.move_speed`
  - `base_stats.base_defense`
  - `status_immunities`
  - `kill_rewards.dice_points`
- 이미 생성된 몬스터에는 정의 변경을 소급 적용하지 않는다. 새로 스폰되는 몬스터부터 적용한다.

## `Tower.json`

- Editor importer가 `TowerDefinition`으로 변환한다.
- 기본 공격력, 사거리, 대상 수, 공격 횟수를 사용한다.
- `base_effect_ids`는 타워 건설 시 런타임 타워에 복사한다.

## `Upgrade.json`

- Editor importer가 `TowerUpgradeDefinition`으로 변환한다.
- `weight`는 선택 확률에 사용한다.
- `stat_modifiers`는 `damage`, `range`, `target_count`, `attack_count`를 지원한다.
- 지원 연산은 `add`, `multiply`, `set`이다.
- `effect_ids.id`는 실행할 효과를 결정한다.
- `effect_ids.value`는 거리, 횟수, 배율, 비율, 쿨다운 등 효과의 실제 수치로 사용한다.
- 기획 수치를 C# 상수보다 JSON 값을 우선하여 사용한다.

## `Tile.json`

- Editor importer가 타일 정의와 36칸 보드 배치를 생성한다.
- 진행 중인 게임 보드에는 소급 적용하지 않고 새 게임부터 반영한다.

# 18. JSON 반영 절차

1. 지정 경로의 JSON을 수정하고 저장한다.
2. Unity가 파일 변경을 감지하면 importer가 자동 실행된다.
3. 자동 반영 여부가 불확실하면 다음 메뉴를 실행한다.
   - `GaeBullBing/Data/Import Monsters JSON`
   - `GaeBullBing/Data/Import Tiles and Towers JSON`
4. Unity Console에서 임포트 성공 로그와 오류를 확인한다.
5. 이미 생성된 런타임 상태가 있는 경우 새 게임을 시작한다.

# 19. 데이터 검증 원칙

- 알 수 없는 몬스터 ID, 타워 ID, 효과 ID는 오류 또는 경고로 보고한다.
- 중복 ID와 비어 있는 ID는 허용하지 않는다.
- 지원하지 않는 스탯 또는 연산은 조용히 무시하지 않는다.
- JSON과 생성된 ScriptableObject가 일치하는지 자동 테스트로 검증한다.
- 동일한 처치에 보상이 중복 지급되지 않아야 한다.
- 결정적인 게임 규칙은 EditMode 테스트로 보호한다.

# 20. QA 배속

- 개발자 콘솔에서 `speed 1`, `speed 2`, `speed 4`, `speed 8` 명령으로 전체 게임 속도를 변경한다.
- 배속은 이동, 투사체, 타워 공격, 몬스터 연출과 시간 기반 대기에 적용한다.
- 새 게임 또는 씬 재시작 시 1배속으로 초기화한다.
